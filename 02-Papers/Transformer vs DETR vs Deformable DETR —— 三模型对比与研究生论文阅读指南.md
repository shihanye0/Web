# Transformer vs DETR vs Deformable DETR —— 三模型对比与研究生论文阅读指南

## 基本信息

|模型|年份|会议|核心思想|
|---|---|---|---|
|Transformer|2017|NeurIPS|纯注意力机制，抛弃 RNN/CNN|
|DETR|2020|ECCV|用 Transformer 做端到端目标检测|
|Deformable DETR|2021|ICLR|可变形注意力 + 多尺度，解决 DETR 缺陷|

---

## 一、模型演进关系

Transformer (2017)  
    ↓ 引入目标检测  
DETR (2020)  
    ↓ 解决收敛慢 + 小物体差  
Deformable DETR (2021)  
    ↓ 后续发展  
DINO DETR / DAB-DETR / RT-DETR ...

**发展脉络**：Transformer 提出了自注意力机制 → DETR 将其应用于目标检测，实现端到端 → Deformable DETR 改进注意力机制，解决 DETR 的实际问题。

## 二、架构对比

### 2.1 Transformer（原始，用于序列到序列任务）

输入序列 → Embedding + 位置编码 → Encoder (自注意力 × 6) → Decoder (自注意力 + 交叉注意力 × 6) → 输出序列

- **任务**：**机器翻译、语言模型等序列任务**
    
- **注意力**：**全局密集注意力（每个 token 与所有 token 交互）**
    
- **解码方式**：**自回归（逐步生成）**
    
- **位置编码**：**正弦余弦函数**
    

### 2.2 DETR（将 Transformer 用于目标检测）

图像 → CNN Backbone → 1×1 卷积降维 → Encoder (自注意力 × 6) → Decoder (自注意力 + 交叉注意力 × 6) → FFN → (类别, 框)  
                                                    ↑                                ↑  
                                              空间位置编码                      Object Query (可学习)

- **任务**：目标检测（集合预测）
    
- **注意力**：**全局密集注意力（每个特征点与所有特征点交互）**
    
- **解码方式**：**非自回归（并行输出 N 个预测）**
    
- **位置编码**：**2D 正弦余弦函数**
    
- **损失函数**：**匈牙利匹配损失（二分图匹配）**
    
- **创新点**：**端到端、无需 NMS、Object Query**
    

### 2.3 Deformable DETR（改进 DETR 的注意力机制）

图像 → CNN Backbone → 多尺度特征图 → Encoder (可变形自注意力 × 6) → Decoder (自注意力 + 可变形交叉注意力 × 6) → FFN → (类别, 框)  
                                              ↑                           ↑  
                                     多尺度位置编码                    Object Query + 参考点  
                               (正弦余弦 + scale-level embedding)

- **任务**：目标检测
    
- **注意力**：**局部稀疏注意力（每个 query 仅与 K 个可学习采样点交互）**
    
- **解码方式**：**非自回归（并行输出）**
    
- **位置编码**：**正弦余弦 + 可学习的 scale-level embedding**
    
- **创新点**：**可变形注意力、多尺度特征、2-stage 模式**

## 三、关键差异对比

### 3.1 注意力机制

|特性|Transformer|DETR|Deformable DETR|
|---|---|---|---|
|类型|全局密集|全局密集|局部稀疏|
|计算复杂度|O(n²)|O((HW)²)|O(HW·K)|
|权重来源|Q·K 点积|Q·K 点积|FC 层直接预测|
|采样位置|所有位置|所有位置|K 个可学习位置|
|适用分辨率|序列长度有限|低分辨率(1/32)|高分辨率(多尺度)|

### 3.2 位置编码

|模型|位置编码方式|特点|
|---|---|---|
|Transformer|正弦余弦函数|固定，可外推到更长序列|
|DETR|2D 正弦余弦函数|固定，仅单尺度|
|Deformable DETR|正弦余弦 + Scale-level Embedding|Scale-level 可学习，区分不同尺度|

### 3.3 检测性能（COCO 数据集）

|模型|AP|AP_S|AP_M|AP_L|训练轮数|
|---|---|---|---|---|---|
|Faster R-CNN (FPN)|37.4|21.2|41.0|49.6|109 epoch|
|DETR|37.7|11.5|39.3|55.3|500 epoch|
|Deformable DETR|40.2|23.2|43.1|55.1|50 epoch|

**关键发现**：

- Deformable DETR 在所有指标上都优于 DETR
    
- 小物体 AP_S 提升最显著（+11.7），得益于多尺度特征
    
- 收敛速度提升 10 倍（500→50 epoch）
    
- 大物体两者接近，DETR 略优（全局注意力的优势）
    

### 3.4 损失函数与后处理

|特性|Transformer|DETR|Deformable DETR|
|---|---|---|---|
|损失函数|交叉熵|匈牙利匹配损失|匈牙利匹配损失|
|是否需要 NMS|N/A|不需要|不需要|
|需要 anchor|N/A|不需要（Object Query）|不需要|
|端到端|N/A|是|是|

## 四、核心知识点总结

### 4.1 从 Transformer 到 DETR 学到了什么

1. **自注意力机制**：让每个位置都能关注全局所有位置，建模长距离依赖
    
2. **编码器-解码器架构**：编码器提取全局特征，解码器基于 query 生成输出
    
3. **并行解码**：目标检测不需要自回归，所有检测可以并行输出
    
4. **端到端思想**：用二分图匹配替代 NMS，简化检测流程
    

### 4.2 从 DETR 到 Deformable DETR 学到了什么

1. **稀疏注意力**：不是所有位置都需要交互，只关注关键采样点即可
    
2. **可学习的采样**：让模型自己决定关注哪些位置，比固定全局注意力更高效
    
3. **多尺度特征**：检测任务需要多尺度，单尺度会遗漏小物体
    
4. **DCN + Transformer 的融合**：结合两者优势（稀疏采样 + 关系建模）
    

### 4.3 研究生应掌握的论文阅读方法

**读论文的三个层次**：

1. **知道它做了什么**：模型架构、输入输出、损失函数
    
2. **知道它为什么这样做**：解决了什么问题、与前人方法的区别、设计动机
    
3. **知道它还能怎么做**：有什么不足、可以怎么改进、与自己研究方向的关联

## 五、与人与人交互行为理解（Human Interaction Understanding）的关联

### 5.1 DETR/Deformable DETR 对交互行为理解的价值

**目标检测是交互行为理解的基础**：要理解人与人之间的交互，首先需要检测出每个人的位置和姿态。

DETR 系列的优势：

1. **端到端检测**：简化检测流程，让研究者专注于交互行为本身
    
2. **关系建模能力**：Transformer 的自注意力天然适合建模人与人之间的关系
    
3. **集合预测思想**：交互行为本身就是一组人之间的关系集合
    

### 5.2 可能的研究方向

**方向 1：基于 Deformable DETR 的多人交互检测**

- 利用可变形注意力的稀疏采样，聚焦于人与人交互的关键区域（如握手、拥抱时的手臂区域）
    
- Object Query 可以专门设计为"交互 Query"，检测交互对而非单个人
    

**方向 2：多尺度特征用于交互行为理解**

- 交互行为涉及不同尺度的信息：手部细节（小尺度）、身体姿态（中尺度）、场景上下文（大尺度）
    
- Deformable DETR 的多尺度可变形注意力天然适合这种需求
    

**方向 3：集合预测用于交互关系建模**

- DETR 的集合预测思想可以扩展：不预测单个物体，而是预测"交互关系集合"
    
- 每个 Object Query 预测一个交互关系（如"推"、"拉"、"握手"），而非单个人
    

**方向 4：结合时序信息**

- DETR/Deformable DETR 处理单帧图像，交互行为理解需要时序信息
    
- 可以将 Deformable DETR 扩展到视频，加入时序注意力（类似 TimeSformer）
    

### 5.3 推荐阅读的后续论文

1. **DAB-DETR**（2022）：用动态 anchor box 替代 Object Query，进一步提升收敛速度
    
2. **DINO DETR**（2022）：引入对比学习和去噪训练，COCO 上首次突破 60 AP
    
3. **RT-DETR**（2023）：实时 DETR，速度媲美 YOLO，精度更高
    
4. **PETR / PETR v2**（2022/2023）：将 DETR 扩展到 3D 目标检测和 BEV 感知
    
5. **MUTR / MOTR**（2021/2022）：将 DETR 扩展到多目标跟踪，适合视频中的交互行为理解
    
6. **QAE-DETR**（2023）：专门用于查询-感知的交互行为检测

### 6.1 人与人交互行为理解（Human Interaction Understanding）的技术栈

图像/视频输入  
    ↓  
目标检测（Deformable DETR / DINO DETR）

### 6.2 可以从 DETR 系列借鉴的设计思想

1. **端到端思想**：交互行为理解也可以做成端到端，不需要复杂的后处理
    
2. **集合预测**：交互关系是集合而非序列，适合用集合预测方式处理

4. **Object Query 设计**：可以设计"交互 Query"，每个 Query 专门检测一种交互类型
    
5. **二分图匹配**：可以扩展为"多方匹配"，将交互参与者与交互类型匹配

# 论文本身收获

## 1. 核心洞察：三篇论文构成一条清晰的"问题→方案→改进"技术链

单独读任何一篇论文，只能看到一个模型的设计。但把 Transformer、DETR、Deformable DETR 放在一起看，能看到一条完整的**技术创新演化路径**：

- **Transformer（2017）**：提出了通用的序列建模原语——自注意力，解决了 RNN 的并行化和长距离依赖问题
    
- **DETR（2020）**：把 Transformer 从序列任务"移植"到视觉任务，证明了端到端检测的可行性，但暴露了收敛慢和小物体差的问题
    
- **Deformable DETR（2021）**：精准识别 DETR 的两个病根，用一个统一设计（可变形注意力）同时治愈两者
    

这种"提出工具→应用工具→改进工具"的三步走模式，是学术研究最经典的推进方式。理解这个模式后，读论文就不再是孤立地理解每个模型，而是看到**技术发展的因果链**。

## 2. 从 Transformer 到 DETR：跨领域迁移的关键不是"照搬"而是"适配"

DETR 不是简单地把 Transformer 塞进检测 pipeline。它做了三个关键适配：

**第一，输入变了。** Transformer 处理一维离散序列（token），DETR 处理二维连续特征图。这要求把 1D 位置编码改为 2D 空间位置编码，把 token embedding 改为 CNN Backbone 特征提取。

- **1D位置编码**：只需要告诉模型**第几个**位置，一个数字就够了。
    
- **2D位置编码**：需要告诉模型在第几**行**、第几**列**，需要两个数字。
    

- |维度|Transformer (原始)|DETR|
    |---|---|---|
    |**处理对象**|一维离散序列 (文本)|二维连续特征图 (图像)|
    |**输入形式**|`[词1, 词2, 词3, ...]`|一个 `H × W` 的网格|
    |**一个“元素”代表**|一个**词** (token)|一个**空间位置** (像素/特征点)|
    |**位置信息**|在一维线上，只需知道是“第几个”|在二维平面上，需知道“第几行，第几列”|
    
    **核心区别**：
    
    - 文本是**一串**，只有前后关系，是一个线性序列。其位置可以用一个整数索引 `i` (0, 1, 2, ...) 唯一确定。
        
    - 图像是**一个面**，有上下、左右关系，是一个二维网格。一个点的位置必须用一对坐标 `(x, y)` 或 `(row, col)` 才能唯一确定。
        

**第二，解码方式变了。** Transformer 是自回归（逐步生成），DETR 是非自回归（并行输出 N 个检测）。这个改变源于对任务本质的洞察：翻译有顺序依赖，检测没有。

**第三，损失函数变了。** Transformer 用交叉熵（每个位置有确定的 target），DETR 用匈牙利匹配（预测和 GT 的对应关系需要动态计算）。这是集合预测任务的本质要求。

这三点适配的启示是：**跨领域迁移架构时，核心不是照搬模块，而是理解新任务的特殊需求，然后针对性地调整。**

## 3. 从 DETR 到 Deformable DETR："诊断"比"开药"更重要

Deformable DETR 的论文最值得学习的不是可变形注意力的设计，而是**对 DETR 问题的精准诊断**：

**收敛慢的根因**：Transformer 注意力初始化时，当 key 元素数量很大（图像像素数），注意力权重趋近均匀分布 1/Nk。模型需要从"关注一切"慢慢学到"只关注重要的"，这个过程非常漫长。

**小物体差的根因**：只用单尺度特征（C5，1/32 分辨率），小物体在特征图上只有几个像素甚至消失。全局注意力无法弥补分辨率不足的问题。

一旦诊断清楚，"药方"就自然了：

- 收敛慢 → 限制每个 query 只看 K 个点（可变形注意力），从一开始就避免均匀注意力
    
- 小物体差 → 引入多尺度特征（FPN 的多个尺度），让小物体在高分辨率层有足够像素
    

这种**"先诊断问题根因，再设计方案"**的研究方法论，比任何具体的模型设计都更有价值。

## 4. 注意力机制的三次演变——从"工具"到"主角"到"被改造"

三篇论文展现了注意力机制角色的三次转变：

|阶段|论文|注意力的角色|关键变化|
|---|---|---|---|
|工具|Transformer (2017)|RNN 的替代品|从辅助变为主角|
|主角|DETR (2020)|检测器的核心引擎|从序列迁移到视觉|
|被改造|Deformable DETR (2021)|被稀疏化、局部化|从全局变局部，从密集变稀疏|

这个演变揭示了一个规律：**任何机制在新领域应用时，都会暴露该领域的特殊问题，从而催生针对性的改造。** Transformer 的全局注意力在序列任务上完美，但在视觉任务上暴露了效率和分辨率问题。Deformable DETR 的改造不是对 Transformer 的否定，而是对特定领域需求的回应。

## 5. 三种注意力的本质区别——不只是"看多少"的问题

三篇论文中的注意力机制看似只是"全局 vs 局部"的区别，但本质差异更深：

**Transformer 的自注意力**：Q、K、V 来自同一序列，权重通过 Q·K 点积计算。含义是"每个 token 和所有 token 的内容相似度"。

**DETR 的交叉注意力**：Q 来自 Decoder（Object Query），K/V 来自 Encoder（图像特征）。含义是"**每个检测目标应该关注图像的哪些区域"。**

**Deformable DETR 的可变形注意力**：没有真正的 K，权重由 Q 通过 FC 层直接预测。含义是"每个检测目标主观认为哪些位置**最重要**"。

从"Q 和 K 的相似度"到"Q 的主观判断"，这是一个根本性的语义转变。标准注意力的权重是**客观的**（由 Q 和 K 的内容决定），可变形注意力的权重是**主观的**（由 Q 自己决定）。这种主观性之所以合理，是因为参考点已经提供了空间先验，Q 只需要在这个先验附近做微调。

## 6. 集合预测思想的通用性——不止于目标检测

DETR 的集合预测思想（固定大小输出 + 二分图匹配）不仅适用于目标检测，还可以推广到任何**输出是无序集合**的任务：

- **全景分割**：输出是一组 (类别, mask) 对
    
- **关键点检测**：输出是一组关键点坐标
    
- **关系检测**：输出是一组 (主体, 关系, 客体) 三元组
    
- **实例分割**：输出是一组 (类别, 框, mask) 三元组
    

DETR 论文本身就展示了这种可扩展性：加一个 mask head 就能做全景分割，PQ=43.4。这种**"一个框架解决多种任务"**的能力，源于集合预测的通用建模方式——它不假设输出的结构，只要求输入输出之间有一一对应关系。

## 7. 消融实验的阅读方法——从对比中理解设计价值

三篇论文的消融实验都做得很好，值得学习其方法论：

**DETR 的消融**：

- 去掉 Encoder → AP 下降 3.9（Encoder 的全局推理很重要）
    
- 去掉辅助损失 → 收敛更慢（深层监督是训练成功的关键）
    
- 去掉 NMS → 性能几乎不变（证明端到端设计有效）
    

**Deformable DETR 的消融**：

- 全局注意力 → 可变形注意力：收敛从 500 epoch 降到 50 epoch
    
- 单尺度 → 多尺度：AP_S 从 11.5 提升到 23.2
    
- 可变形注意力 + FPN vs 可变形注意力 alone：无提升（说明可变形注意力已覆盖 FPN 功能）
    

消融实验的核心原则：**每次只改一个变量，观察性能变化。** 如果去掉某个模块后性能大幅下降，说明该模块重要；如果变化不大，说明该模块可能是冗余的。读消融实验比读模型描述更能理解每个设计的真正价值。

## 整合重点：从"知道它们好"到"知道它们为什么好"

### 一句话总结

三篇论文的核心价值不在于各自的模型设计，而在于它们共同展示了一个**技术从理论到应用再到优化的完整生命周期**：通用工具（Transformer）→ 跨领域应用（DETR）→ 针对性改进（Deformable DETR）。理解这个生命周期，就掌握了技术研究的基本方法论。

### 我认为最重要的三个 takeaway

**1. 技术创新的"三步走"模式**

Transformer 提出工具 → DETR 证明可行性 → Deformable DETR 解决实际问题。这不是偶然的三篇论文，而是**技术创新的通用模式**：先有通用方法，再有领域应用，最后有针对性改进。在自己的研究中，找到自己处于哪一步——是在提出新工具、证明新应用、还是解决已有方法的问题——决定了论文的定位和写法。

**2. "问题驱动"比"方法驱动"更有价值**

DETR 的问题意识是"检测器依赖手工组件"，Deformable DETR 的问题意识是"DETR 收敛慢且小物体差"。两者都是**从现有方法的具体问题出发**，而不是"我有一个新方法，看看能用在哪里"。问题驱动的研究更容易找到创新点，因为问题是具体的、可量化的，而方法是抽象的、需要验证的。

**3. 跨领域迁移需要"翻译"而非"照搬"**

DETR 成功的关键不是"用了 Transformer"，而是**把 Transformer 的概念翻译成了检测任务的语言**：序列→特征图、token→像素、自回归→并行输出、交叉熵→匈牙利匹配。每一次翻译都需要对两个领域都有深刻理解。在自己的研究中，如果要借鉴其他领域的方法，先想清楚：源领域的什么概念对应目标领域的什么概念？

### 一句话反驳"这就是三个独立的模型"

表面上看是三个独立的模型，但它们共享同一条技术脉络：**注意力机制的演化**。Transformer 定义了注意力的计算范式（QKV + softmax），DETR 把它应用到视觉域并引入集合预测，Deformable DETR 把它从全局密集改为局部稀疏。每一步都是对前一步的**直接回应**：Transformer 的问题是"只能处理序列"→ DETR 回应"也能处理图像"→ DETR 的问题是"收敛慢、小物体差"→ Deformable DETR 回应"用可变形注意力解决"。这不是巧合，而是技术发展的必然路径。
