# 现有项目可复用与不可复用部分

## 可以复用的部分

| 现有资产 | 路径 | 实机价值 | 使用方式 |
|---|---|---|---|
| π0.5 训练配置与数据格式 | `isaac_g1/outputs1/pi05_final_scene_train_scripts/`、LeRobot 数据集 | 可继续训练真实采集数据 | 保持 28D 合同，但在数据入口和 DDS 出口使用同一个显式 adapter |
| π0.5 HTTP 推理服务 | `success_final/server/pi05_act_server_finalscene_1000.py` | 可将 Python 3.12 的 π0.5 与其他环境解耦 | 作为 A6000 的推理服务参考；保留 `/health`、输入形状与 NaN 校验 |
| checkpoint processor | checkpoint 内 `policy_preprocessor` / `policy_postprocessor` | 保证训练与推理的归一化一致 | 部署时必须随 checkpoint 一起加载，不能手写归一化 |
| 动作安全锚定思路 | `success_final/README_FINAL.txt` | VLA residual + 专家锚点比裸控更适合早期实机试验 | 先用经过实机验证的保持位姿/轨迹作 anchor，再增加小范围 residual |
| Dex3 模拟 DDS 包装 | `isaac_g1/assets/unitree_sim_isaaclab_min/dds/dex3_dds.py` | 可参考官方话题和 `HandCmd_`/`HandState_` 类型 | 只借鉴消息形状，实机程序应直接基于官方 SDK 新建 |
| G1 模拟 DDS 包装 | `isaac_g1/assets/unitree_sim_isaaclab_min/dds/g1_robot_dds.py` | 可参考 `LowState_`、`LowCmd_`、CRC 的处理概念 | 只作协议学习，不可直接作为实机控制节点 |

## 不可直接上机的部分

| 资产                                      | 证据                                         | 原因                                    | 正确处理                 |
| --------------------------------------- | ------------------------------------------ | ------------------------------------- | -------------------- |
| `success_final/run_final_demo.sh`       | 激活 `isaaclab45`，进入 `isaac_g1/pysimscripts` | 启动的是 Isaac Lab，不是 G1 实机程序             | 保留作仿真回归，不部署到机器人      |
| `final_scene_real_g1dex3_*.py`          | 导入 `isaaclab.app`、`omni.usd`、`pxr`         | 名称虽含 `real_g1dex3`，实际创建 USD 场景        | 禁止直接用于真机             |
| `dds/dex3_dds.py`、`dds/g1_robot_dds.py` | 通过共享内存 `isaac_*` 与仿真交换数据                   | 数据方向是“DDS <-> Isaac”，不是“DDS <-> 真实硬件” | 以官方 SDK 创建新的硬件网关     |
| 当前 28D 手部标签                             | 已发现标签与原始数值列顺序不一致                           | 会导致手指命令错位风险                           | 固化并单测双手重排索引，再做逐轴实机确认 |
| 20 步 SGD smoke checkpoint               | 仅 20 次 update                              | 只验证训练链路，未证明任务能力                       | 不作为实机控制候选模型          |

## 建议新增的实机 owner

不要改造某个 `final_scene_*` 脚本来承载实机语义。应单独创建一个硬件网关模块，作为唯一 owner：

```text
camera + G1/Dex3 DDS state
  -> 28D adapter（顺序/单位/方向）
  -> π0.5 HTTP 推理服务
  -> safety gate（新鲜度、限幅、速度、限位、急停、watchdog）
  -> G1 LowCmd + Dex3 HandCmd DDS
```

其中只有最后一跳接触真实机器人。HTTP 服务只输出 checkpoint 后处理后的 28D 动作，不拥有实机关节顺序、限位或安全策略。

## 最小验收

实机网关首次实现后，至少要具备：输入输出 28D 形状检查、每关节名称表、左右手重排单元测试、命令新鲜度超时、NaN 拒绝、位置/速度/增量限幅、急停、状态和动作日志。任何一项缺失都不应进入实际动作测试。
