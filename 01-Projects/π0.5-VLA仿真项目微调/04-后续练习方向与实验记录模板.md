# 后续练习方向与实验记录模板

后续练习应按“先保证合同正确，再研究效果，最后研究部署”的顺序推进。一次只改变一个变量，否则无法判断结果由什么造成。

## 推荐练习阶梯

| 级别 | 练习 | 主要问题 | 验收标准 |
|---|---|---|---|
| 1 | 复现阶段 D | 我能否独立跑通训练/保存/重载？ | 新 checkpoint、日志、单样本和 21 样本评估齐全 |
| 2 | 参数与日志阅读 | 每个参数实际改变哪条链路？ | 能从 `train_config.json` 反推启动配置和可训练参数 |
| 3 | 28D 合同修复 | 每个数值列到底对应哪个关节？ | source -> dataset -> DDS 映射逐列测试通过 |
| 4 | 独立验证 | 模型是在记忆训练集还是能泛化？ | episode 级 holdout，stats 只用训练集生成 |
| 5 | 单变量训练实验 | steps、seed、增强分别有什么影响？ | 同协议重复评估，有明确基线和停止条件 |
| 6 | LoRA + AdamW | 能否在 12GB 上更接近正式 optimizer？ | trainable 参数和 optimizer state 可控，无 OOM，可重载 |
| 7 | Isaac 闭环 | 离线 MAE 是否转化为任务成功？ | 固定场景和扰动集上的重复成功率 |
| 8 | 实机 adapter | 数值如何安全到达 G1/Dex3？ | 只读、单关节低速、限位/急停/watchdog 分级通过 |

## 方向一：先练训练工程，不追效果

用现有脚本完成一次自己的 D 阶段闭环。只记录事实：路径、参数、loss、显存、产物和重载。当前错误手部标签不影响你学习训练机制，但输出不得接 Dex3。

停止条件：你能不看旧结论，自己从日志解释 693M/4.14B、SGD、scheduler 自动缩放和 checkpoint 内容。

## 方向二：修复数据与机器人合同

推荐保留原始手部列顺序，修正元数据名称，并在 adapter 建立显式 DDS 重排。每只手当前源顺序到 DDS 顺序的局部索引为 `[2,5,6,1,4,0,3]`。

需要的自动测试：

1. 28 个 source name 唯一且长度正确。
2. dataset state/action/name 使用同一 canonical order。
3. dataset -> DDS 和 DDS -> dataset 双向 round-trip 无损。
4. 左右手分别测试，禁止共享错误的 URDF 顺序假设。
5. residual 映射长度必须为 28，不能再被 14 项 `zip` 静默截断。

仅修正标签、不改变数值列时，旧 checkpoint 可以按索引重新评估；若重排数据列，则必须重训或增加等价置换层。

## 方向三：建立真正的验证集

当前六模型评估使用训练 episode，MAE 下降只能说明拟合。下一步应按 episode 划分，不能随机拆帧：相邻帧高度相似，随机拆帧会数据泄漏。

优先核查未进入训练的 `episode_20260703_171539_00932frames` 是否独立、是否成功、是否由已有轨迹派生。只有来源闭合后才能作为 holdout。

验收：训练 stats 不包含 holdout；checkpoint 选择依据是预先固定的 holdout 指标，而不是最后一步编号。

## 方向四：单变量训练实验

建议顺序：

1. `seed=1000` 与另一个 seed：观察短训练波动和重复性。
2. 20 步与 50/100 步：只研究工程趋势，不宣称效果。
3. 图像增强关闭/开启：在独立验证集上看视觉扰动鲁棒性。
4. 不同 episode holdout：检查模型是否只适应某一条轨迹。
5. 数据量增加：比较 7 条与更多成功演示，而不是只延长训练。

每轮只改一个变量，并固定数据版本、基础模型、评估样本和 seed。没有独立验证前，不做大规模超参数搜索。

## 方向五：LoRA 与低显存正式 optimizer

这应晚于 D 阶段复现和 28D 合同修复。目标不是“让 LoRA 跑起来”，而是验证它能否用 AdamW 在 12GB 上形成可信基线。

需要先检查：

- 实际 trainable parameter names/count，而不是只看 `Using PEFT!`。
- PI05 默认 PEFT target 是否覆盖预期的 expert q/v 与动作投影。
- `time_mlp_in/out` 当前未被默认 target 明确覆盖，是否需要加入。
- adapter checkpoint 能否独立加载，processor 是否随模型保存。
- 与 expert-only SGD、历史 AdamW 的比较协议是否一致。

不要直接上 QLoRA、8-bit optimizer、CPU offload 三项组合。一次引入一个机制，失败才可定位。

## 方向六：从离线指标进入仿真

离线 MAE 通过后，再恢复 Isaac Lab：先固定场景，随后改变物体位置、光照和相机。记录任务成功率、失败阶段、动作 TTL、NaN/Inf 拒绝和服务退出降级。

当前 residual 右手未真正写入，因此在修复 28 项映射前，不能把仿真结果解释为 Dex3 VLA 控制效果。

## 三个月后最可能失败的原因

- 一直优化训练集 MAE，没有独立 episode 或仿真成功率。
- 修了标签但没有唯一 canonical order 和自动映射测试。
- 为适配 12GB 同时引入多种低显存技术，结果不可归因。
- checkpoint 很多但没有哈希、配置、数据版本和选择依据。
- 仿真链路继续复用陈旧动作或 14 项映射截断，离线模型再好也无法正确执行。

因此每个阶段都要有停止条件：合同不闭合不谈效果，验证不独立不选 checkpoint，仿真不安全不接实机。
