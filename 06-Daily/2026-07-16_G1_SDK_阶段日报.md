实际作用：现在计算机可以把两条固定中文命令交给 G1；同一进程中同一请求被网络重传时，系统返回第一次结果而不重复发出 TTS。该能力为后续服药提醒、状态播报和任务确认提供可靠的非运动通信底座。

## 今日收获

- SDK 闭环不只看“RPC 返回 0”，还必须同时看实体状态、播放状态、网络稳定性与进程关闭顺序。
- 中文命令不等于开放式理解；当前是确定性映射到两个冻结 intent，便于逐步扩展且不混入硬件语义。
- `request_id` 的作用是识别一次任务请求而非限制用户再次下令：要再次播报只需发送新的 ID；同 ID 是同一任务的重传。
- `rt/audio_msg` 当前只有 `play_state` 的实机证据，没有已验证的请求 ID 关联字段，因此只能报告“请求后观察到播放”，不能宣称严格的消息因果关联。
