# 命理运势系统 - 项目手册

> 更新日期：2026-06-24

## 项目简介

基于中国传统命理学（八字、周易六爻、奇门遁甲）的个性化运势推算系统。用户注册后填写生辰信息，系统每日自动推算运势并通过 QQ 邮箱/飞书推送。

**访问地址**：http://82.157.186.52

## 功能清单

| 功能      | 说明                     |
| ------- | ---------------------- |
| 用户注册/登录 | 邮箱注册，JWT 认证            |
| 每日运势    | 八字排盘 → 运势推算 → LLM 解读   |
| 时辰运势    | 12 个时辰的宜忌分析            |
| 概率事件    | 今日可能发生事件及概率            |
| 六爻占卜    | 手动摇卦，LLM 解读卦象          |
| 奇门遁甲    | 问题起盘 / 实时看盘            |
| 每日推送    | QQ 邮箱 + 飞书 Webhook     |
| 个人中心    | 生辰修改、推送设置、LLM 配置、准确率统计 |

## 技术栈

### 前端
- Vue 3 + TypeScript + Vite 6
- Element Plus UI 组件库
- Pinia 状态管理
- Vue Router 路由

### 后端
- Python 3.10 + FastAPI
- SQLAlchemy ORM + SQLite
- JWT 认证 + bcrypt 密码哈希
- Xiaomi MiMo API（LLM 解读）

### 部署
- 腾讯云 Ubuntu 22.04（2核2G）
- Nginx 反向代理 + 静态文件服务
- systemd 进程管理
- GitHub Actions CI/CD

## 推送系统

### 推送流程

```
GitHub Actions（每小时触发）
  → POST /api/v1/internal/daily-push
    → 查找 push_time 匹配当前小时的用户
    → 计算八字 + 运势 + 时辰运势 + 概率事件
    → LLM 生成运势解读
    → QQ 邮箱 / 飞书 Webhook 推送
```

### 推送内容

- 综合运势评分 + 四维评分（事业/财运/感情/健康）
- 幸运色/数字/方位
- 运势解读（LLM 生成，400-600 字）
- 12 时辰运势（表格展示，含宜忌）
- 今日概率事件（带概率百分比）

### 用户配置

用户在个人中心可以：
- 开启/关闭每日推送
- 选择推送渠道（QQ邮箱 / 飞书 / 两者）
- 设置推送时间（默认 07:00）
- 配置飞书 Webhook URL

## LLM 配置

### 供应商信息

| 项目 | 值 |
|------|------|
| 供应商 | Xiaomi MiMo |
| API URL | `https://token-plan-cn.xiaomimimo.com/v1` |
| 模型 | `mimo-v2.5` |
| 格式 | OpenAI 兼容（`/chat/completions`） |

### 配置层级

```
用户个人中心配置（优先）
  ↓ 未配置时回退
服务器 .env 配置（默认）
```

### 注意事项

- API URL 不能用 `/anthropic`，必须用 `/v1`
- 模型名含 `[1M]` 等标注时会自动清理
- API Key 脱敏显示（含 `***`），保存时自动跳过脱敏值

## 常见问题

### 1. 网站打不开

```bash
# 检查后端是否运行
sudo systemctl status fortune

# 检查 Nginx 是否运行
sudo systemctl status nginx

# 检查端口是否被占用
sudo netstat -tlnp | grep 8080
```

### 2. 登录报错 500

```bash
# 查看后端日志
sudo journalctl -u fortune --no-pager -n 30

# 常见原因：数据库表不存在
cd /opt/fortune/server
sudo venv/bin/python -c "from app.models.base import Base; from app.models import *; from app.config import settings; from sqlalchemy import create_engine; engine = create_engine(settings.DATABASE_URL); Base.metadata.create_all(engine)"
sudo systemctl restart fortune
```

### 3. 推送不工作

```bash
# 手动触发测试
curl -X POST http://localhost:8080/api/v1/internal/daily-push \
  -H "Content-Type: application/json" \
  -d '{"hour": 13}'

# 检查 SMTP 配置
cd /opt/fortune/server
sudo venv/bin/python -c "import smtplib; from app.config import settings; s=smtplib.SMTP_SSL(settings.SMTP_HOST,settings.SMTP_PORT); s.login(settings.SMTP_USER,settings.SMTP_PASSWORD); print('OK')"
```

### 4. 内存不足

```bash
# 添加 1GB swap
sudo fallocate -l 1G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

### 5. Git 拉取失败（国内服务器）

```bash
# 使用镜像
git remote set-url origin https://gitclone.com/github.com/shihanye0/fortune.git
git pull origin main
```
