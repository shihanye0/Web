# 具身智能评估方向论文阅读顺序

## 背景

今天已读完：

- [[Embodied-Reasoner - Synergizing Visual Search Reasoning and Action for Embodied Interactive Tasks]]

接下来不建议马上跳到另一个具身方法论文，而是先补“评估框架”这一条线。原因是我当前正在跑 `/home/user/github-product/embodied_reasoner` 项目的评估任务，最需要理解的是：多轮交互 agent 怎么评估、轨迹质量怎么看、失败类型如何拆分、benchmark 如何设计。

## 推荐阅读顺序

### 1. Natural-Language Agent Harnesses.pdf

优先读这篇。

它最贴合当前阶段。Embodied-Reasoner 本质上也是一个 agent harness：环境给 observation，模型输出 thought/action，环境返回 feedback，然后进入下一轮。读这篇的目标是理解如何搭建可重复、可比较、可诊断的 agent 测试环境。

重点关注：

- harness 是什么；
- 多轮 agent 如何被评估；
- 任务、环境、模型、日志、指标之间如何解耦；
- 为什么只看最终成功率不够。

读完后回头看 Embodied-Reasoner 项目的 `evaluate` 目录，会更容易理解。

### 2. General Modular Harness for LLM Agents in Multi-Turn Gaming Environments.pdf

第二篇读这个。

它和 Embodied-Reasoner 都属于多轮交互评估，只是环境从具身房间换成了游戏环境。适合用来比较不同环境下的 state/action/reward 或 observation/action/feedback 如何设计。

重点关注：

- 多轮任务如何记录轨迹；
- 如何判断 agent 是真的会规划，还是碰巧成功；
- failure case 如何分类；
- 游戏环境评估和 AI2-THOR 具身评估有哪些共通点。

### 3. Beyond Accuracy: Behavioral Testing of NLP Models with CheckList.pdf

第三篇读这个。

这篇不是具身智能论文，但它提供很重要的“评估思维”。读它的目的不是学 NLP 测试本身，而是学习如何把模型能力拆成可测试行为，而不是只看一个总分。

可以对应到 Embodied-Reasoner 的行为维度：

- 是否重复搜索；
- 是否忽略历史观察；
- 是否输出非法动作；
- 是否提前终止；
- 是否无法处理隐藏物体；
- 是否在组合任务中丢失子目标。

### 4. VL-CheckList: Evaluating Pre-trained Vision-Language Models with.pdf

第四篇读这个。

Embodied-Reasoner 是视觉-语言-动作模型，视觉理解是前置能力。VL-CheckList 可以帮助拆分视觉语言能力，例如对象识别、属性理解、关系理解、空间理解。

读的时候要联系失败案例：

- 找不到物体，是视觉识别失败，还是搜索策略失败？
- 放错位置，是空间关系理解失败，还是动作规划失败？
- 重复搜索，是记忆失败，还是反思失败？
- 非法动作，是可交互性理解失败，还是环境约束理解失败？

### 5. SkillsBench: Benchmarking How Well Agent Skills Work Across Diverse Tasks.pdf

第五篇读这个。

Embodied-Reasoner 的动作可以看成多个 agent skill 的组合：搜索、打开、拾取、放置、验证、结束。SkillsBench 能帮助从“技能粒度”理解 agent 能力，而不是只看整条任务是否成功。

重点关注：

- skill 是否可迁移；
- skill 成功率如何评估；
- 多技能组合为什么容易失败；
- benchmark 如何覆盖不同任务类型。

## 最小阅读计划

如果时间有限，只读前三篇：

1. `Natural-Language Agent Harnesses.pdf`
2. `General Modular Harness for LLM Agents in Multi-Turn Gaming Environments.pdf`
3. `Beyond Accuracy: Behavioral Testing of NLP Models with CheckList.pdf`

这三篇读完，应该能明显提升我对 Embodied-Reasoner 项目评估代码、评估日志和失败案例的理解。
