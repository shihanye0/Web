## 一句话总结

**女娲造人**：输入人名/主题/模糊需求，自动深度调研 → 思维框架提炼 → 生成可运行的人物 Skill。让乔布斯、马斯克、芒格、费曼都给你打工。

---

## 核心理念

女娲不是复制人，是**提炼思维框架**。一个好的人物 Skill 是一套可运行的认知操作系统：

- 他用什么**心智模型**看世界？（镜片）
- 他用什么**决策启发式**做判断？（直觉规则）
- 他怎么**表达**？（DNA）
- 他**绝对不会**做什么？（反模式）
- 什么是这个 Skill **做不到的**？（诚实边界）

**关键区分**：捕捉的是 HOW they think，不是 WHAT they said。

## 使用方式

### 一、直接使用已有 Skill

在 Claude Code 对话中直接说：

```
用芒格的视角帮我分析这个投资决策
费曼会怎么解释量子计算？
乔布斯会怎么看这个产品设计？
切换到马斯克，我想讨论降本方案
用张雪峰的视角帮我分析专业选择
```

**退出角色**：说「退出」「切回正常」「不用扮演了」即可恢复正常模式。

### 二、用女娲蒸馏新人物

```
蒸馏一个张小龙
造一个段永平的视角 Skill
帮我做一个雷军的 Skill
```

**模糊需求触发**：
```
我想提升决策质量，有没有合适的思维方式？
有没有一种思维框架能帮我看透商业本质？
我需要一个思维顾问帮我做人生选择
```

女娲会诊断你的需求，推荐 2-3 个候选人物，然后自动蒸馏。

### 三、更新已有 Skill

```
更新乔布斯的 Skill，最近有新动态
马斯克又收购了一家公司，更新一下
```

## 女娲的工作原理

### 执行流程

```
用户输入人名
    ↓
Phase 0: 入口分流
    ├─ 明确人名 → 直接路径（Phase 0A）
    └─ 模糊需求 → 诊断路径（Phase 0B）→ 推荐候选 → 用户选择
    ↓
Phase 0.5: 创建 Skill 目录
    ↓
Phase 1: 6个并行 Agent 调研
    ├─ Agent 1: 著作/长文 → 01-writings.md
    ├─ Agent 2: 播客/访谈 → 02-conversations.md
    ├─ Agent 3: 社交媒体 → 03-expression-dna.md
    ├─ Agent 4: 他人评价 → 04-external-views.md
    ├─ Agent 5: 决策记录 → 05-decisions.md
    └─ Agent 6: 人生时间线 → 06-timeline.md
    ↓
Phase 1.5: 调研质量 Review 检查点
    ↓
Phase 2: 框架提炼
    ├─ 心智模型提取（3-7个，三重验证）
    ├─ 决策启发式提取（5-10条）
    ├─ 表达 DNA 分析
    ├─ 价值观与反模式
    └─ 诚实边界
    ↓
Phase 2.5: 提炼确认检查点
    ↓
Phase 3: 构建 SKILL.md
    ↓
Phase 4: 质量验证
    ├─ 已知测试（Sanity Check）
    ├─ 边缘测试（Edge Case）
    └─ 风格测试（Voice Check）
    ↓
Phase 5: 双 Agent 精炼
    ↓
交付
```

### 三重验证筛选

一个观点要被收录为心智模型，必须同时通过三项验证：

| 验证 | 说明 |
|------|------|
| **跨域复现** | 在 ≥2 个不同领域/话题中出现过（不是随口一说） |
| **生成力** | 能推断此人对新问题的立场（有预测力） |
| **排他性** | 不是所有聪明人都会这么想（有独特性） |

三重通过 → 心智模型；仅 1-2 重 → 降级为决策启发式；0 重 → 丢弃。

## 信息源优先级

| 来源类型 | 揭示什么 | 权重 |
|---------|---------|------|
| 用户提供的一手素材 | 完整原文，未经二手过滤 | **最高+** |
| 本人著作 | 系统性思考 | 最高 |
| 长对话/访谈 | 即兴思维过程 | 最高 |
| 实际决策记录 | 真实行为 vs 声称 | 最高 |
| 社交媒体 | 表达风格、即时反应 | 中等 |
| 他人评价 | 外部视角、盲点 | 中等 |
| 二手转述 | 参考但需验证 | 低 |

**信息源黑名单**（永远排除）：知乎、微信公众号、百度百科/百度知道。

## 效果示例

### 示例 1：Naval 视角

```
用户      ❯ 我同时想做自媒体、写书、做独立开发，精力完全不够

Naval     ❯ 你列了三个欲望。每个欲望都是你跟不快乐签的一份合同。
            不是精力不够，是合同太多。
            
            问你自己：哪一个做起来你会忘记时间？
            那个就是你的特定知识所在。
            
            不是三选一。是先一，再一，再一。
            串行复利，不是并行内耗。
```

### 示例 2：马斯克视角

```
用户      ❯ 我们的SaaS产品获客成本太高了

马斯克     ❯ 先别想怎么降。先算物理极限。
            获客的最小必要动作是什么？用户从知道你到付费，
            信息传递的理论最短路径是什么？
            
            现在你的实际路径是理论路径的几倍？
            如果超过3倍，中间一定有可以砍掉的步骤。
            不是优化漏斗，是质疑漏斗本身该不该存在。
```

### 示例 3：乔布斯视角

```
用户      ❯ OpenAI和Anthropic谁的方向是对的？

乔布斯    ❯ 你问错了。这不是一场关于方向的竞赛。
            这是一场关于品味的竞赛。
            
            OpenAI像1985年的微软。什么都做，什么都要赢。
            Anthropic更像早期的Apple。聚焦。
            Focus means saying no to a hundred good ideas.
            
            但两家公司都犯了一个我绝对不会犯的错误——
            他们不控制硬件。
            最终赢的可能是同时控制芯片、模型和用户界面的人。
            你知道现在谁在同时做这三件事吗？Apple.
```

每个蒸馏完成的 Skill 都是自包含的：

```
~/.claude/skills/[person-name]-perspective/
├── SKILL.md                          # 核心技能定义
├── scripts/                          # 工具脚本（如有）
└── references/
    └── research/                     # 调研数据
        ├── 01-writings.md            # 著作与系统思考
        ├── 02-conversations.md       # 长对话与即兴思考
        ├── 03-expression-dna.md      # 碎片表达与风格DNA
        ├── 04-external-views.md      # 他者视角与批评
        ├── 05-decisions.md           # 决策记录与行动
        └── 06-timeline.md            # 人物时间线
```

## 诚实边界（局限性）

每个 Skill 都明确标注做不到什么：

- **蒸馏不了直觉** — 框架能提取，灵感不能
- **捕捉不了突变** — 截止到调研时间的快照
- **公开表达 ≠ 真实想法** — 只能基于公开信息

> 一个不告诉你局限在哪的 Skill，不值得信任。
