## Q1："你现在了解多少？"

### 灵巧手接口已学透

Dex3-1 每只手 3 根手指、7 个电机，通过 DDS 协议通信，已整理完整接口文档（见 [[Unitree Dex3-1 灵巧手 DDS 通信接口文档]]）。控制结构体 MotorCmd_ 的 6 个参数都清楚：

| 参数 | 含义 |
|------|------|
| mode | 0=锁定，1=FOC 控制 |
| q | 目标位置（rad） |
| dq | 目标速度（rad/s） |
| tau | 前馈力矩（N·m） |
| kp | 位置刚度（Nm/rad） |
| kd | 速度阻尼（Nm/(rad/s)） |

PD 控制器公式：`输出力矩 = kp × (q_target - q_actual) + kd × (0 - dq_actual) + tau`

触觉传感器：12 路压力传感器，`pressure[i] >= 100000` 为有效数据，`30000` 为无数据。

### 目标检测有基础

之前学过 DETR 和 Deformable DETR 论文，Transformer 架构也比较熟，药瓶识别可以快速上手。

- 力控算法和触觉传感器数据处理也需要进一步学习

## Q2："有什么想法？"

### 技术方案框架

**三阶段推进：**

| 阶段 | 时间 | 目标 | 产出 |
|------|------|------|------|
| 第 1 阶段 | 1-2 周 | 跑通 SDK，实现基本抓取 | 能编译运行示例代码，能抓取简单物体 |
| 第 2 阶段 | 2-3 周 | 开发"辅助服药"核心算法 | 视觉感知+运动规划+力控抓取 |
| 第 3 阶段 | 1-2 周 | 比赛优化 | 速度、成功率、安全性提升 |

**第 2 阶段三个模块：**

1. **视觉感知**：用 DETR/Deformable DETR 检测药瓶位置，输出 bounding box + 类别
2. **运动规划**：正/逆运动学 + 轨迹插值，规划机械臂从当前位置到药瓶的平滑轨迹
3. **力控抓取**：触觉传感器监测接触力，自适应调整 kp/kd，多阶段抓取（软接触→检测→加力→稳定）

### 关键技术难点

| 难点 | 解决思路 |
|------|----------|
| 力控抓取稳定性 | 不同材质不同力，用触觉反馈做自适应力控 |
| 视觉-力觉融合 | 视觉粗定位 + 触觉精定位 + 抓取确认 |
| 实时性要求 | 状态反馈 1000 Hz，控制命令 100 Hz，DDS 发布-订阅 |
