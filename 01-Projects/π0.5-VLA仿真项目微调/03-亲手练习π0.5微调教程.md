# 亲手练习 π0.5 微调教程

本教程让你亲手完成一次“检查命令 -> 20 步训练 -> checkpoint 检查 -> 独立推理评估”的闭环。当前数据手部名称尚未修正，因此本教程只练工程流程，产出的模型不得直接用于 Dex3。

## 你需要什么

- Conda 环境：`lerobot312`。
- GPU：RTX 4070 12GB，当前主机已验证。
- 基础模型：`/home/user/zhangxu/models/pi05_base`。
- 数据集和现有本机 smoke 脚本。
- 至少约 10GB 新磁盘空间；当前每个完整 checkpoint 约 8.8GB。

## 第 1 步：只打印命令，不训练

```bash
DRY_RUN=1 /home/user/zhangxu/isaac_g1/outputs1/pi05_final_scene_train_scripts/run_pi05_final_scene_smoke20_local.sh
```

你应看到 `[PROJECT_ROOT]`、`[DATASET_ROOT]`、`[MODEL_ROOT]`、`[OUT_DIR]` 和完整 `[COMMAND]`。先逐项解释命令中的数据、冻结策略、optimizer 和输出目录；解释不清楚时不要开始训练。

## 第 2 步：检查 GPU 和磁盘

```bash
nvidia-smi
df -h /home/user/zhangxu
```

确认没有其他高显存进程，并预留至少 10GB。训练前 GPU 不是空闲状态时先停下，避免把环境竞争误判为模型 OOM。

## 第 3 步：运行自己的 20 步 smoke

```bash
/home/user/zhangxu/isaac_g1/outputs1/pi05_final_scene_train_scripts/run_pi05_final_scene_smoke20_local.sh
```

脚本自动创建带时间戳的新目录，不覆盖历史 checkpoint。模型加载约 80 秒，训练主体约几十秒；终端最终应打印 `End of training`、`DONE`、`OUT_DIR` 和 `LOG`。

把下面等号右侧替换为脚本最后实际打印的完整 `OUT_DIR`，再执行：

```bash
RUN_DIR=/home/user/zhangxu/isaac_g1/outputs1/pi05_final_scene_7demo_smoke20_local_sgd_2026XXXXXXXXXX
```

## 第 4 步：检查训练和 checkpoint

```bash
cat "$RUN_DIR/checkpoints/000020/training_state/training_step.json"
du -sh "$RUN_DIR/checkpoints/000020"
find "$RUN_DIR/checkpoints/000020" -maxdepth 3 -type f
```

预期：`step=20`，目录约 8.8GB，至少包含模型权重、config、preprocessor、postprocessor 和 training state。只看到训练进程退出、不检查这些产物，不算完成。

再检查日志中的关键证据。日志路径在脚本结束时打印：

```bash
rg 'num_learnable_params|step:20|Checkpoint policy|End of training|CUDA out of memory' 你的日志路径
```

应看到 693,422,112 个可训练参数、第 20 步、保存和训练结束，不应出现 OOM。

## 第 5 步：先做一个样本的独立重载

为评估建立新的输出目录，不覆盖阶段 C。先把日期时间改成本次实际值：

```bash
EVAL_DIR=/home/user/zhangxu/isaac_g1/outputs1/practice_eval_你的时间戳

PYTHONDONTWRITEBYTECODE=1 PYTHONNOUSERSITE=1 \
HF_HOME=/home/user/zhangxu/hf_cache \
HF_HUB_OFFLINE=1 TRANSFORMERS_OFFLINE=1 HF_DATASETS_OFFLINE=1 \
/home/user/miniconda3/envs/lerobot312/bin/python \
/home/user/zhangxu/isaac_g1/outputs1/eval_pi05_final_scene_fixed_samples/eval_fixed_samples.py \
  --models practice \
  --custom-model "practice=$RUN_DIR/checkpoints/000020/pretrained_model" \
  --sample-limit 1 \
  --output-dir "$EVAL_DIR"
```

预期看到 `All keys loaded successfully!`、一条 `[SAMPLE 01/01]` 和 `failures=0`。

## 第 6 步：完成 21 个固定样本评估

确认单样本通过后，去掉 `--sample-limit 1`，换一个新的 `EVAL_DIR` 再运行。检查：

```bash
cat "$EVAL_DIR/summary.json"
```

记录 `mae_mean`、`peak_memory_gb`、`latency_ms_mean`、`repeat_max_abs_diff`。这些样本来自训练数据，只用于检查推理完整性，不能作为泛化指标。

## 第 7 步：写实验记录

使用 [[04-后续练习方向与实验记录模板]]，至少填写：

- 你运行的完整命令。
- 只改了哪个变量。
- 训练和评估输出路径。
- 峰值显存、loss 范围、MAE 和失败信息。
- 本次结论能证明什么、不能证明什么。

## 常见问题

### tokenizer 仍指向 `/data/zhangxu`

说明加载了旧配置。当前本机基础模型和新 checkpoint 应指向 `/home/user/zhangxu/models/paligemma-3b-pt-224_tokenizer`。

### AdamW 第 1 步 OOM

这是已复现的 optimizer state 显存不足，不是 NVIDIA 驱动故障。不要靠反复重跑解决；回到 SGD smoke，或另立 LoRA + AdamW 实验。

### 训练完成但 MAE 几乎不变

20 步 SGD 的目标是工程闭环，不是效果。不要擅自增加到 1000 步；先修复 28D 合同并建立独立验证协议。

### 想删除练习 checkpoint 节省空间

先确认实验记录、日志、`summary.json` 和关键哈希已保存，再单独确认要删除的时间戳目录。不要使用通配符批量删除历史正式 checkpoint。

## 你完成了什么

完成本教程后，你应能独立解释并操作：训练入口、参数冻结、optimizer、loss 日志、checkpoint 结构、processor 和独立重载。这比“进程跑完”更接近真正掌握模型微调。
