# DETR - End-to-End Object Detection with Transformers

## 基本信息

- **作者**: Nicolas Carion, Francisco Massa, Gabriel Synnaeve, Nicolas Usunier, Alexander Kirillov, Sergey Zagoruyko (Facebook AI)
- **文件**: DETR 2005.12872v3
- **会议**: ECCV 2020
- **添加时间**: 2026-05-10 10:03

## 摘要

提出 DETR（DEtection TRansformer），将**目标检测视为直接的集合预测问题**。通过基于集合的全局损失（二分图匹配强制唯一预测）和 Transformer 编码器-解码器架构，移除了 NMS、anchor 等手工设计组件，**实现端到端检测**。在 COCO 数据集上与高度优化的 Faster R-CNN 性能相当，大物体检测更优，且可扩展到全景分割任务。

# 预备知识：
## 1.二分图匹配
二分图匹配是一种**最优分配算法**，用来决定100个预测框中，**哪个框去匹配哪个真实物体**（哪些框应该被惩罚、哪些框应该被认为是"没找到物体"）。
DETR一次性输出**100个预测框**（固定数量）。
一张图片里可能只有**3个真实物体**（比如猫、狗、车）。
**问题来了**：这100个预测框中，哪些应该去匹配猫？哪些去匹配狗？剩下的97个怎么办？
### 传统方法的问题

Faster R-CNN 用**anchor + NMS**：

- 每个anchor独立判断"IoU > 0.5就是正样本"
    
- 可能导致**多个anchor匹配同一个物体**（需要NMS去重）
    
- 需要手工设计的anchor尺寸、比例
    

**DETR想做的**：让模型**自己学会**每个query负责检测哪个物体，不依赖NMS。

## 2.object query
**一句话定义**：Object Query 是 DETR 中**一组可学习的向量**（数量固定，如100个），每个向量负责"寻找"并输出一个物体。
无论输入什么图像，这100个Query向量都**不变**。

那它们怎么检测不同图像中的不同物体？  
→ 通过 **Transformer 解码器的交叉注意力**：Query 去图像特征中"询问"、提取信息。
┌─────────────────────────────────────────────────────────┐
│  Step 1: 初始化                                         │
│  Object Queries = [Q1, Q2, ..., Q100]  (形状: 100×512)  │
│  (可学习的参数，随机初始化)                              │
└─────────────────────────────────────────────────────────┘
                          ↓

│    - 自注意力: Q之间互相"聊天"（避免重复检测）            │
│    - 交叉注意力: Q去图像特征中"寻找"物体                  │
│    - FFN: 深度思考                                       │
└─────────────────────────────────────────────────────────┘

│    - 边界框 (x, y, w, h)                                │
└─────────────────────────────────────────────────────────┘
### “向量不变”指的是什么？

这里的“不变”，指的是 **这组向量参数在模型训练完成之后，就被固定了。** 无论你给模型一张猫的图，还是一张车的图，这100个向量 (`[v1, v2, ..., v100]`) 作为解码器的输入，其**具体的数值是不变的**。

- 可以把它理解为：模型预先准备好了100个 **“探测器”** 。这100个探测器本身是固定的。
    
- **类比**：就像100个拿着不同“任务卡”的侦探。
    
    - 侦探A的任务卡上写着：“寻找红色的圆形物体”
        
    - 侦探B的任务卡上写着：“寻找左上角的大型物体”
        
    - 这些“任务卡”在出发前就制定好了，不会因为今天是去查找“火灾现场”还是“盗窃现场”而改变。
### 2. 那模型如何适应不同图像？

虽然这100个**向量参数**不变，但它们在**解码器每一层计算时的状态**（也就是通过注意力机制“看到”图像后的结果）是**变化的**。

过程是这样的：

1. **初始化**：解码器接收这100个固定的、静态的向量。
    
2. **交叉注意力（“看图”）**：每个固定的向量，都会去“看”编码器从当前图像中提取的特征图。
    
    - `侦探A`（任务：找红色圆物）看当前图像——如果是番茄（红色圆），注意力权重就高；如果是蓝天（蓝色），注意力权重就低。
        
    - `侦探B`（任务：找左上角物体）看当前图像——如果图像主体在左上角，它就活跃；如果在右下角，它就沉默。
        
3. **向量被动态更新**：当固定向量“看到”图像的具体内容后，它会**加权融合**图像特征，从而**更新自己**，变成一个**包含了当前图像信息的、新的向量**。
    

**所以，不变的是“任务卡”（初始Query向量），变化的是“侦察报告”（经过交叉注意力后的Query）。**

## 一、DETR 要解决什么问题

### 1.1 目标检测任务

输入自然图像，输出其中每个感兴趣对象的**边界框（bounding box）和类别标签**。

### 1.2 现有检测器的痛点

现有检测器（无论 one-stage 还是 two-stage）都是用**间接方式**处理集合预测：

- **Two-stage**（如 Faster R-CNN）：先用 Region Proposal 生成候选区域，再对每个候选做分类+回归
- **One-stage**（如 YOLO、FCOS）：基于 anchor 或物体中心点做密集预测

它们共同的问题：

1. **依赖 anchor / proposal 等先验知识**：需要手工设计 anchor 的大小、长宽比、铺设方式
2. **依赖 NMS 后处理**：同一物体会产生大量重叠的预测框，需要用 NMS 去重
   - NMS 引入额外超参数（阈值等），调参复杂
   - NMS 不是所有硬件都支持，部署困难
3. **性能受限于初始猜测的设计**：研究表明，最终性能与 anchor/proposal 的设定方式高度相关

### 1.3 DETR 的思路

**将目标检测直接视为集合预测问题，用端到端网络直接输出预测集合。**

核心想法：
- 一张图的 GT（真实标签） 就是一个 (类别, 框) 的集合
- DETR 输出一个固定大小 N=100 的预测集合
- 通过**二分图匹配**找到预测与 GT 的最优一一对应
- 匹配上的算检测损失，匹配不上的预测为"无物体"（∅）
- **不需要 NMS，不需要 anchor，真正的端到端**

## 二、DETR 的核心贡献

1. **端到端的检测框架**：移除了 NMS、anchor 等人工先验知识，训练和部署更简单
2. **基于二分图匹配的集合预测损失**：强制模型输出一组**独一无二**的预测（每个物体理想状态只生成一个框）
3. **Transformer 编码器-解码器架构**：利用全局建模能力，一次传递预测所有对象并建模对象间关系
4. **Object Query 机制**：可学习的位置编码，类似"可学习的 anchor"，限定模型输出 N 个预测
DETR 不依赖像Faster R-CNN那样根据图像内容动态生成的区域建议（Proposals）作为Query起点，而是**完全依赖一套固定的、可学习的“任务分配”机制**，这种设计更简洁，但需要模型自己学会如何用固定的Query去匹配多样的图像内容。
5. **可扩展到全景分割**：只需加一个 mask head 即可统一处理 stuff 和 thing 类别

## 三、DETR 模型架构

整体架构由三个组件组成：**CNN Backbone → Transformer 编码器-解码器 → 预测 FFN**

### 3.1 CNN Backbone

- 输入图像 x_img ∈ R^{3×H₀×W₀}
- 使用 ResNet-50 提取特征，得到低分辨率激活图 f ∈ R^{C×H×W}，典型值 C=2048, H=W=H₀/32
- 用 1×1 卷积将通道维度从 C 降到 d（如 d=256），得到 z₀ ∈ R^{d×H×W}

### 3.2 Transformer Encoder

- 将 z₀ 的空间维度展平为一维，得到 d×HW 的序列输入：把特征图从"二维网格"拉直成"一维列表"，让Transformer能够像处理句子一样处理图像。每个"像素位置"变成了序列中的一个"词"，2048维的特征向量就是这个词的"嵌入"。
- 标准 Transformer Encoder 结构：多头自注意力 + FFN，共 N=6 层
- **添加固定的空间位置编码**（2D 正弦余弦编码），加入到每一层注意力的输入中
- Encoder 的作用：**进一步学习全局信息**，为 decoder 的预测做铺垫

### 3.3 Transformer Decoder

- 与标准 Transformer Decoder 的关键区别：**N 个对象检测是并行解码的**，不是自回归一个一个输出
- 输入：
  - **Object Queries**：N 个可学习的位置编码（N=100），类似"可学习的 anchor"
  - Encoder 的输出（图像特征）
- 结构：多头自注意力 + 编码器-解码器交叉注意力 + FFN，共 M=6 层
- 输出：N 个 d 维嵌入向量，每个独立送入预测 FFN

**为什么 Object Query 必须不同？** 因为 Decoder 也是排列不变的（permutation-invariant），如果 N 个输入嵌入相同，输出也会相同，无法产生不同的检测结果。

|维度|含义|类比|
|---|---|---|
|**固定**|训练完成后参数不再改变，**与输入图像无关**|100个侦察兵是固定编制，不会今天100明天200|
|**不同**|这100个向量之间**互不相同**，各有自己的值|每个侦察兵有自己的名字和任务，不是克隆人|

### 3.4 预测 FFN

- **分类**：3 层 MLP（多层感知机）（ReLU 激活，隐藏维度 d）+ 线性投影 + Softmax，预测类别标签
- **回归**：3 层 MLP + 线性投影，预测归一化的 (center_x, center_y, width, height)
- **特殊类别 ∅**：表示"无物体"，相当于背景类。N 通常远大于实际物体数，大部分 slot 预测为 ∅

### 3.5 每个 Decoder 层的辅助损失

DETR 的一个独特设计：**在每个 Decoder 层之后都添加预测 FFN 和匈牙利损失**，而不是只在最后一层。

DETR 在每个Decoder层后都加上预测头和损失函数，让每一层都直接参与学习，就像每个年级都要参加期末考试。这样不仅能加速训练、稳定梯度，还能实现"从粗到细"的迭代优化效果。

- **所有预测 FFN 共享参数**
    
- 使用额外的**共享 LayerNorm** 来规范化来自不同 Decoder 层的 FFN 输入
    
- 作用：帮助模型在每一层都产生有意义的预测，加速训练收敛，类似 deep supervision

## 四、集合预测损失（匈牙利损失）

### 4.1 核心问题

DETR 输出 N=100 个预测框，但一张图的 GT 可能只有几个。**如何匹配？如何计算 loss？**

### 4.2 二分图匹配（Bipartite Matching）

将问题转化为：在所有 N! 种可能的一一对应（排列）中，找到总代价最小的那个。

**匹配代价函数** L_match(y_i, ŷ_σ(i))：

```
L_match = -𝟙{c_i ≠ ∅} · p̂_σ(i)(c_i) + 𝟙{c_i ≠ ∅} · L_box(b_i, b̂_σ(i))
```

- c_i：GT 的类别标签，b_i：GT 的边界框
- p̂_σ(i)(c_i)：预测 σ(i) 对应类别 c_i 的概率
- b̂_σ(i)：预测 σ(i) 的边界框
- 𝟙{c_i ≠ ∅}：只对非空（非背景）GT 计算代价

**最优匹配**通过**匈牙利算法**高效求解：

```
σ̂ = argmin_{σ∈S_N} Σ L_match(y_i, ŷ_σ(i))
```

**匈牙利算法的直觉**：类似"腾地方"——先来的给后来的腾地方，递归操作，最终找到全局最优的一一配对。

### 4.3 匈牙利损失（Hungarian Loss）

找到最优匹配 σ̂ 后，计算训练损失：

```
L_Hungarian = Σ [-log p̂_σ̂(i)(c_i) + 𝟙{c_i ≠ ∅} · L_box(b_i, b̂_σ̂(i))]
```

与匹配代价的区别：
- 分类部分使用 **log 概率**（匹配代价用的是原始概率，为了让分类和框的代价量级可比）
- **∅ 类的 log 概率权重除以 10**，以平衡正负样本比例（类似 Faster R-CNN 的正负样本采样）

### 4.4 边界框损失 L_box

使用 **L1 损失 + 广义 IoU（GIoU）损失**的线性组合：

```
L_box = λ_iou · L_IoU(b, b̂) + λ_L1 · ||b - b̂||₁
```

- λ_L1 = 5, λ_iou = 2
- 为什么需要 GIoU？纯 L1 损失对大框和小框的尺度不一致（同样相对误差，大框的绝对值更大），GIoU 是尺度不变的
- 所有损失按 batch 内的物体数量归一化

## 五、关键概念详解

### 5.1 非自回归（Non-autoregressive）与并行解码

**自回归**（如原始 Transformer 翻译）：逐步生成，第 t 步的输出依赖第 t-1 步。推理必须串行，速度慢。

**非自回归/并行解码**（DETR 的方式）：N 个检测结果**一次性并行输出**。

DETR 选择非自回归的原因：
- 目标检测中，**物体之间没有严格的生成顺序**——检测一只猫不需要先检测一辆车
- 并行输出比串行更快
- 配合二分图匹配损失，保证每个预测有唯一对应，不会有重复

### 5.2 排列不变性（Permutation Invariance）

**问题**：DETR 并行输出 N 个预测，这些预测的顺序是任意的。如果预测 [A, B] 和 [B, A] 算出不同的 loss，模型就不知道该学什么了。

**解决**：匈牙利匹配保证了——无论预测以什么顺序输出，都能找到与 GT 的最优一一对应，计算出的 loss **对预测的排列不变**。

这就是为什么损失函数"通过预测的排列而保持不变"：对所有可能的排列 σ 搜索最优匹配，loss 只取决于匹配结果，不取决于预测的原始顺序。

### 5.3 NMS（非极大值抑制）

**作用**：从大量重叠的预测框中选出最佳的一个。

**过程**：
1. 按置信度排序所有预测框
2. 选得分最高的框，删除与它 IoU 超过阈值的其他框
3. 重复直到没有框剩余

**DETR 为什么不需要 NMS？** 因为匈牙利匹配强制每个 GT 只对应一个预测，模型学会不产生重复检测。实验验证：在 Decoder 第一层加 NMS 有小幅提升（因为第一层还没充分建模对象间关系），但后面几层 NMS 几乎无用。

### 5.4 Anchor 与 Proposal

**Anchor**：预先定义的一组大小、长宽比不同的参考框，密集铺在图像上。模型不直接预测坐标，而是预测每个 anchor 相对于真实物体的偏移量。

**Proposal**：从 anchor 中筛选并精调后得到的、可能包含物体的区域（two-stage 模型的中间结果）。

**DETR 的做法**：直接预测绝对坐标（相对于整张图），不使用 anchor 偏移量。

### 5.5 Object Query

DETR 中的 Object Query 是**可学习的位置编码**（learned positional embeddings），作用类似于"可学习的 anchor"：

- 数量固定为 N=100
- 每个 Query 通过注意力机制与全局图像特征交互
- 每个 Query 学会专注于不同的位置和物体大小（论文 Fig.7 可视化）
- 训练后，不同 Query 会自然分化出不同的"专长"（有的擅长检测小物体，有的擅长检测图像级别的大框）

### 5.6 优化器：SGD、Adam、Adagrad

DETR 论文中使用的优化器细节：

**DETR 实际使用 AdamW**（Adam with decoupled weight decay）：
- 学习率 Backbone: 10⁻⁵，Transformer: 10⁻⁴
- Backbone 学习率比 Transformer 小一个数量级，这对稳定训练很重要
- 权重衰减 10⁻⁴
- 梯度裁剪：最大梯度范数 0.1

三种常见优化器的对比：

| 优化器 | 核心思想 | 优点 | 缺点 |
|--------|---------|------|------|
| **SGD** | 随机梯度下降，沿梯度方向更新，可加动量 | 简单，泛化性好，很多经典模型用 SGD | 需要仔细调学习率，收敛慢 |
| **Adam** | 结合动量（一阶矩）和自适应学习率（二阶矩），每个参数有独立学习率 | 收敛快，对学习率不敏感 | 泛化性可能不如 SGD，内存占用大 |
| **Adagrad** | 根据历史梯度平方和自适应调整学习率，频繁更新的参数学习率变小 | 适合稀疏数据 | 学习率会单调递减，后期可能过小导致停止学习 |

AdamW 是 Adam 的改进版：将权重衰减从梯度更新中解耦出来（原始 Adam 的 L2 正则化与权重衰减不等价），训练效果更好。

### 5.7 辅助损失（Auxiliary Decoding Loss）

**DETR 在每个 Decoder 层之后都计算匈牙利损失**，而不是只在最后一层计算。

为什么要这样做？
- Decoder 有 6 层，如果只在最后一层算 loss，前面几层的梯度信号很弱
- 辅助损失让每一层都被监督，加速收敛
- 类似于 deep supervision（深度监督）的思想
- 实验表明这对 DETR 的训练至关重要（DETR 训练需要 500 epoch，比普通检测器长很多）

实现细节：
- **所有层的预测 FFN 共享参数**（不是每层一个独立 FFN）
- 使用**额外的共享 LayerNorm** 规范化不同层 FFN 的输入

## 六、DETR 的优缺点

### 优点

1. **大物体检测效果好**：得益于 Transformer 的全局建模能力，不受 anchor 大小限制
2. **端到端、无需 NMS**：训练和部署更简单
3. **架构简单**：不需要特殊库，几百行代码就能实现
4. **可扩展性强**：加 mask head 即可做全景分割

### 缺点

1. **小物体检测效果差**：没有 FPN（特征金字塔网络）、没有多尺度特征、没有复杂的检测头
2. **训练极慢**：需要 500 epoch（COCO 上大多数模型只需几十个 epoch）
3. **固定 N=100 的限制**：不能预测超过 100 个物体（实际够用，但理论上有限制）

## 七、实验结果

### 7.1 与 Faster R-CNN 的对比

- COCO val 上 DETR（ResNet-50）AP=37.7，与 Faster R-CNN（FPN, 9× schedule）的 37.4 相当
- 大物体 AP_L：DETR 显著优于 Faster R-CNN（+5.5）
- 小物体 AP_S：DETR 较差（-5.3）
- DETR-DC5（带空洞卷积的版本）在各尺度上都更好

### 7.2 Encoder/Decoder 层数的影响

- 不使用 Encoder 时 AP 下降 3.9，大物体下降 6.0 → Encoder 的全局推理很重要
- Decoder 层数越多性能越好（Fig.4）

### 7.3 全景分割

- 加 mask head 后，DETR 在 COCO 全景分割上 PQ=43.4（R50），优于 PanopticFPN
- 在 stuff 类别上尤其强（得益于 Encoder 的全局注意力）
