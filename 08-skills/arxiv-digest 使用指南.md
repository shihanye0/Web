# arxiv-digest 使用指南

## 概述

arxiv-digest 是一个面向科研人员的 arXiv 每日论文情报引擎。它每天从 arXiv 拉取指定领域的最新论文，用 AI 完成筛选、解读和中文摘要，把原本半小时到一小时的论文巡检压缩成几分钟内可读完的研究日报。

它不是简单的论文列表聚合器，而是一套轻量级研究情报工作流：脚本负责获取真实数据，AI 负责判断研究价值、提炼方法贡献、梳理技术趋势。

**核心价值**：不是"今天发了什么"，而是"今天最值得你注意什么"。

## 使用示例

### 基本用法

```
你：帮我看看今天 arXiv 上计算机视觉有什么新论文
```

arxiv-digest 会：
1. 调用脚本从 arXiv 抓取最近 2-3 天的 CV 领域论文
2. AI 筛选出最值得读的 5 篇左右
3. 生成一份中文研究日报，包含趋势总览和论文亮点

### 指定领域

```
你：生成一份 NLP 方向的 arXiv 中文日报
```

### 保存日报

```
你：把今天的 arXiv 日报保存下来
```

日报会保存到 `~/arxiv-digest/YYYY-MM-DD-领域.md`

## 手动运行脚本

不依赖 AI 时，也可以在终端里直接调用抓取脚本：

```bash
python3 ~/.claude/skills/arxiv-digest/scripts/fetch_arxiv.py --field cv --days 2 --max 30
```

### 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--field` | 研究领域代码 | 必填 |
| `--days` | 抓取最近几天的论文 | 2 |
| `--max` | 最多拉取多少篇候选论文 | 30 |

### 使用建议

- **周一**：设 `--days 3-4`，把周末的论文也带上
- **日常巡检**：`--days 2 --max 30` 是合理配置
- **深度调研**：可以增大 `--max` 到 50-100

## 输出格式

一份典型日报会包含：

1. **趋势总览**：今天这个领域大致有什么趋势/热点
2. **精选论文**（约 5 篇）：
   - 中文亮点标题（点明这篇做了什么）
   - 2-3 句话的"人话摘要"（解决什么问题、用了什么思路、结果如何）
   - 原文链接
3. **被筛掉的论文**：不用解释，日报要精炼

### 输出风格

- 像同行帮你划重点，不要堆术语
- 多用"这篇解决了……""相比之前的方法，它……"这样的句式
- 一份日报控制在一屏能读完
- 强调"可决策"：帮助你判断哪篇要精读、哪篇要收藏

## 定时自动运行

### 配置定时任务

```bash
# 每天早上 8:30 自动拉取 cv 领域
python3 ~/.claude/skills/arxiv-digest/scripts/setup_autorun.py --field cv --hour 8 --minute 30
```

### 平台支持

| 平台 | 底层机制 | 注意事项 |
|------|---------|---------|
| macOS | launchd（用户级 LaunchAgent） | 睡眠时不触发，唤醒后等下次整点 |
| Linux | crontab | 最稳定，服务器部署首选 |
| Windows | Task Scheduler | 第一次运行可能需要管理员权限 |

### 删除定时任务

```bash
python3 ~/.claude/skills/arxiv-digest/scripts/setup_autorun.py --remove
```

### 输出位置

结果每天自动写入 `~/arxiv-digest/logs/output.json`，AI 可直接读取并生成日报。

## 适合谁

- 每天需要跟踪 arXiv 新论文的研究生、博士后和高校教师
- 想快速捕捉技术趋势的 AI 工程师、算法团队和产品研究团队
- 正在做文献综述、开题、选题调研或竞品技术监测的人
- 希望把论文筛选流程沉淀为日常机制的实验室或研究小组

## 最佳实践

- **日常巡检**：每天花 2-3 分钟读日报，快速判断哪篇值得精读
- **周一补读**：周一设 3-4 天，把周末的论文也覆盖
- **定期复盘**：每周花 10 分钟回顾本周日报，梳理技术趋势
- **组会分享**：把精选论文分享给团队，作为选题参考

## 设计理念

arxiv-digest 的核心不是"更快地列出更多论文"，而是"更早地发现更重要的论文"。它把重复性的抓取流程交给脚本，把需要研究判断的部分交给 AI，让研究者把注意力放回真正有价值的事情上：提出问题、比较方法、设计实验，以及决定下一步读什么。
