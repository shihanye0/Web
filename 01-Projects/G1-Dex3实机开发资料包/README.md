# G1-Dex3 实机开发资料包

本目录是机器人到位后要同步到 A6000、并作为实机开发入口阅读的精简资料包。它不复制模型权重、数据集、账号配置或仿真输出；这些大文件应按版本单独同步和校验。

## 先读什么

1. [[01-官方接口与到机验收]]：官方真源、DDS 话题、到机后的只读验收。
2. [[02-现有项目可复用与不可复用部分]]：当前仿真项目中哪些组件可作为实机开发基础，哪些不能直接上机。
3. [[03-A6000开发部署边界]]：本地、A6000 和机器人侧控制机各自负责什么。
4. [[04-A6000同步清单]]：首次同步代码、环境、数据和模型时的边界与验收。

## 最重要的结论

- G1/Dex3 的官方 SDK、实际关节 ID、模式切换、限位、错误码和固件兼容性是实机真源；不要用仿真 USD 或旧实验脚本替代。
- 当前 π0.5 项目的训练与 HTTP 推理服务可以复用，但 `success_final` 下名称含 `real_g1dex3` 的脚本仍在 Isaac Lab 内运行，不能直接控制真机。
- 现有 28D 数据合同的双臂 14 维与官方 G1 SDK 的电机 ID `15..28` 可对应；双手 14 维存在元数据标签与原始数值列顺序不一致的问题。未建立显式 adapter 并完成逐轴测试前，禁止向真机发送模型动作。
- A6000 应成为代码、训练环境、模型和数据的主工作站；真正发送 DDS 低层动作的安全网关应靠近机器人，运行官方 SDK 并独立限幅、急停和 watchdog。

## 不应放入此目录或同步到服务器的内容

- 密码、API key、`auth.json`、浏览器 Cookie、SSH 私钥、完整的 `~/.codex` 或 `~/.claude` 登录状态。
- 未筛选的 `outputs/`、缓存、视频原始副本和多份 checkpoint。
- 未经过实机安全审查的控制脚本。
