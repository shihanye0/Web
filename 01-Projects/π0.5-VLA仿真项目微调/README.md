# π0.5 VLA 仿真项目微调

这个目录用于学习“阶段 D 到底训练了什么，以及如何亲手练习 π0.5 微调”。它把原理、参数、操作教程和实验记录分开，避免把一次跑通当成模型有效。

## 建议阅读顺序

1. [[01-阶段D微调机制与结果]]：先理解阶段 D 实际更新了哪些参数、loss 如何产生。
2. [[02-阶段D配置参数参考]]：逐项查训练参数及其影响。
3. [[03-亲手练习π0.5微调教程]]：从 dry-run 到新 checkpoint 重载完成一次闭环。
4. [[04-后续练习方向与实验记录模板]]：按风险从低到高安排后续实验。
5. [[养老照护大赛-π0.5-VLA微调项目接管指南]]：回到整个项目的 A-E 阶段主线。

## 一句话结论

阶段 D 是一次 **20 步、batch size 1、动作专家微调、无 momentum SGD 的工程 smoke**。它更新约 6.934 亿参数，冻结约 34.50 亿 PaliGemma VLM 参数；它证明训练、保存、processor 和重载链路可运行，但没有证明效果、泛化或实机可用性。

## 当前必须保留的边界

- 手部元数据标签与原始数值列顺序不一致；当前模型学习的是稳定的“索引列”，不能按错误标签直接下发 Dex3。
- 当前 `use_relative_actions=false`，关节名称没有参与 loss。可以保留现有列顺序、修正名称并在 adapter 显式重排到 DDS；只有改变训练列顺序时才必须重训。
- 当前 residual 脚本的 `arm28_order` 只有 14 项，右手 VLA residual 实际没有写入仿真。
- 每次 20 步练习会保存约 8.8GB 的完整模型，不要无计划重复生成。

## 最推荐的学习主线

```text
看懂 D 阶段配置
  -> dry-run 复述完整命令
  -> 亲手复现 20 步工程 smoke
  -> 独立进程评估自己的 checkpoint
  -> 学会只改一个变量并记录实验
  -> 修复 28D 名称/adapter 合同
  -> 建立 episode 级 holdout
  -> 再研究 LoRA + AdamW 或更大显存正式训练
```
